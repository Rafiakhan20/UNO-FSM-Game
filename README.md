# GUI-based UNO Card Game Simulation using FSM

This Python project, developed for the Automata Theory course, simulates the popular UNO card game using Finite State Machine (FSM) principles. Each phase of gameplay—Initial Turn, Card Drawn, Wild Color Selection, and Terminal State—is modeled as a distinct state, with player actions like Play, Draw, and Pass acting as input symbols that trigger deterministic transitions. This approach demonstrates a practical application of FSM in interactive software design.

The game is built with Object-Oriented Programming (OOP) for structured modeling of cards, players, decks, and game logic. Tkinter provides an interactive GUI, while PIL handles dynamic card rendering, ensuring accurate display of colors, numbers, and special cards. Real-time GUI updates reflect player moves, turn changes, and special actions.

All UNO rules are implemented, including Skip, Reverse, Draw Two, Wild, and Wild Draw Four cards. The game supports multiple players, dynamic deck shuffling, controlled turn passing, and direction changes. Wild and Wild Draw Four cards allow proper color selection, and a custom winner animation provides clear game completion feedback.

This project combines theoretical FSM concepts with practical software implementation, showcasing structured, rule-based game logic, real-time interaction, and modular design that can be extended or modified easily.
